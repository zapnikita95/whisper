# Как создать командный файл (shell script) на Mac

## Что это такое?

**Командный файл** на Mac — это `.command` файл (или `.sh`). Аналог `.bat` на Windows.

**Важно:** `.command` файлы можно запускать **двойным кликом** в Finder — они автоматически откроются в Terminal.app. `.sh` файлы нужно запускать из терминала.

## Как создать

### Вариант 1: В терминале (рекомендуется для .command)

1. Открой **Терминал** на Mac
2. Перейди в папку с проектом:
   ```bash
   cd ~/Desktop/whisper
   ```
3. Создай файл:
   ```bash
   nano start-client.command
   ```
   (или используй другой редактор: `vim`, `code` и т.д.)

4. Вставь команды (см. примеры ниже)
5. Сохрани: в `nano` — `Ctrl+O` (Enter), затем `Ctrl+X`
6. Сделай файл исполняемым:
   ```bash
   chmod +x start-client.command
   ```
7. **Для .command:** просто двойной клик в Finder!  
   **Для .sh:** запусти из терминала: `./start-client.sh`

### Вариант 2: В текстовом редакторе

1. Открой **TextEdit** или любой редактор
2. Создай новый файл
3. Вставь команды
4. Сохрани как `start-client.command` (важно: расширение `.command`)
5. В терминале сделай исполняемым:
   ```bash
   chmod +x start-client.command
   ```
6. **Двойной клик в Finder** — откроется Terminal и выполнится!

**Разница:**
- `.command` — можно запускать двойным кликом из Finder
- `.sh` — нужно запускать из терминала (`./script.sh`)

## Структура shell скрипта

### Базовый пример

```bash
#!/bin/bash
# Это комментарий — начинается с #

# Команды выполняются построчно:
echo "Привет, мир!"
python3 my-script.py
```

### Важные элементы

1. **Шебанг** (первая строка):
   ```bash
   #!/bin/bash
   ```
   Говорит системе, какой интерпретатор использовать.

2. **Комментарии**:
   ```bash
   # Это комментарий
   ```

3. **Переменные**:
   ```bash
   SERVER_URL="http://100.115.68.2:8000"
   echo "Сервер: $SERVER_URL"
   ```

4. **Условия**:
   ```bash
   if [ ! -f "file.txt" ]; then
       echo "Файл не найден"
       exit 1
   fi
   ```

5. **Параметры командной строки**:
   ```bash
   # Передаются через $1, $2, $3...
   # Все параметры: "$@"
   python3 script.py "$@"
   ```

## Примеры

### Пример 1: Простой запуск

```bash
#!/bin/bash
cd ~/Desktop/whisper
python3 whisper-client-mac.py --server http://100.115.68.2:8000
```

### Пример 2: С проверками

```bash
#!/bin/bash
# Проверка Python
if ! command -v python3 &> /dev/null; then
    echo "Ошибка: Python 3 не найден"
    exit 1
fi

# Проверка файла
if [ ! -f "whisper-client-mac.py" ]; then
    echo "Ошибка: файл не найден"
    exit 1
fi

# Запуск
python3 whisper-client-mac.py --server http://100.115.68.2:8000
```

### Пример 3: С параметрами

```bash
#!/bin/bash
SERVER_URL="${1:-http://100.115.68.2:8000}"  # $1 или значение по умолчанию
LANGUAGE="${2:-ru}"  # $2 или "ru"

python3 whisper-client-mac.py --server "$SERVER_URL" --language "$LANGUAGE"
```

Использование:
```bash
./script.sh http://192.168.1.100:8000 en
```

### Пример 4: Интерактивный

```bash
#!/bin/bash
read -p "Введи IP сервера: " SERVER_IP
read -p "Язык (ru/en): " LANG

python3 whisper-client-mac.py --server "http://$SERVER_IP:8000" --language "$LANG"
```

## Полезные команды

- `cd "$(dirname "$0")"` — перейти в папку со скриптом
- `command -v python3` — проверить, установлен ли Python 3
- `[ ! -f "file.txt" ]` — проверить, существует ли файл
- `"$@"` — все параметры командной строки
- `"${VAR:-default}"` — значение переменной или значение по умолчанию
- `read -p "Вопрос: " VAR` — спросить у пользователя

## Частые ошибки

1. **Permission denied** — не сделал файл исполняемым:
   ```bash
   chmod +x script.sh
   ```

2. **Command not found** — не указал путь к команде:
   ```bash
   # Плохо:
   python script.py
   
   # Хорошо:
   python3 script.py
   # или
   /usr/bin/python3 script.py
   ```

3. **Файл не найден** — неправильный путь:
   ```bash
   # Используй абсолютные пути или cd в нужную папку
   cd "$(dirname "$0")"
   ```
