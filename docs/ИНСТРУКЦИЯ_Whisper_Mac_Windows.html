Whisper: диктовка с Mac на Windows-сервер

Цель: на Windows крутится сервер с GPU, на Mac — клиент в строке меню. Нажимаешь сочетание клавиш, говоришь, текст вставляется в активное окно.

Всего несколько шагов на каждой системе — без ежедневного Терминала.

Часть A — Windows (сервер)

  1. Скачай установщик. Открой в браузере страницу релизов репозитория на GitHub (раздел Releases). Скачай файл вида WhisperSetup-… .exe (самый свежий релиз).
  2. Установи. Дважды щёлкни по файлу → «Далее» / «Next» → при необходимости согласись с правами администратора → дождись окончания → «Готово».
  3. Запусти Whisper Server. Через меню «Пуск» найди ярлык Whisper Server и открой его. Должно появиться окно с номером порта и подсказкой про сеть.
  4. Брандмауэр. Если Windows спросит разрешение для сети — нажми «Разрешить» для частной сети (и при использовании Tailscale тоже разреши, если спросит).
  5. Запиши адрес для Mac. В окне будет строка вида http://127.0.0.1:ПОРТ. Для Mac с другого компьютера нужен IP этого ПК в той же сети или в Tailscale (в окне часто есть подсказка с Tailscale IPv4). Запиши что-то вроде http://100.x.x.x:ПОРТ — это вставишь/проверишь на Mac один раз (или Mac сам найдёт сервер, если настроен сканер портов).
Проверка обновлений (Windows): после запуска GUI через несколько секунд приложение само сравнивает версию с GitHub. Если есть новее — появится вопрос «Скачать обновление?». Согласись — откроется загрузка установщика. Ничего вручную искать не обязательно.
Эталон голоса на сервере (редко нужно): если на сервере включена проверка голоса, эталонный файл speaker_embedding.npy должен лежать в профиле пользователя Windows (см. README про WHISPER_SPEAKER_VERIFY). В обычном сценарии достаточно эталона только на Mac.

Часть B — Mac (клиент)

  1. Скачай и поставь приложение. На GitHub в Releases скачай WhisperClient-… .dmg. Открой DMG → перетащи Whisper Client в папку Программы (Applications).
  2. Первый запуск и права. Открой Программы → запусти Whisper Client. Если macOS ругается на разработчика: Системные настройки → Конфиденциальность и безопасность → «Всё равно открыть».
  3. Мониторинг ввода. Системные настройки → Конфиденциальность и безопасность → Мониторинг ввода — включи переключатель для Whisper Client (или Python, если так отображается). Без этого горячие клавиши не сработают.
  4. Микрофон. В том же разделе «Конфиденциальность» открой Микрофон и разреши для Whisper Client.
  5. Иконка в строке меню. Справа сверху должна появиться иконка 🎤 (или значок приложения). Нажми на неё: там версия, адрес сервера, Проверить обновления, запись эталона голоса и выход.
  6. Как пользоваться. Зажми сочетание ⌃ ⌥ ⇧ (Control + Option + Shift), говори, отпусти все три — пойдёт распознавание и вставка в то приложение, которое было активно в начале записи.
Проверка обновлений (Mac): примерно через 10–15 секунд после старта клиент сам проверяет GitHub. Если вышла новая версия — придёт уведомление. Полное скачивание DMG — через пункт меню 🎤 → Проверить обновления….
Эталон голоса (Mac) — без Терминала: меню 🎤 → Записать эталон голоса (45 с)… → «Начать» → говори 45 секунд одним голосом. После сохранения чужие голоса не отправляются на сервер.

Один раз установить модули (если меню эталона пишет про ошибку): открой «Терминал» (Программы → Утилиты), вставь одну строку и нажми Enter:
python3 -m pip install torch resemblyzer
(это 1–2 ГБ загрузки, нужен интернет). Потом снова открой пункт эталона в меню.
После первого сохранения эталона при следующих запусках Whisper Client из .app проверка голоса включается сама (файл лежит в ~/.whisper/speaker_embedding.npy). Отключить только через правку run.sh внутри .app или переменную WHISPER_MAC_NO_SPEAKER_VERIFY=1 — для обычного пользователя не нужно.

Открыть этот файл в Word

В Microsoft Word: Файл → Открыть → выбери этот HTML-файл. При необходимости Сохранить как → Документ Word (.docx).

Актуальные детали и «если не работает» — в README.md репозитория. Лог на Mac: ~/Library/Logs/WhisperMacClient.log (фильтр в Консоли: WHISPER_MAC).
